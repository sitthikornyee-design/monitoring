# Project Overview

This project is a website for monitoring KPIs and business performance.

## Goal
Build a clean, modern, easy-to-use website that helps users:
- view KPI summaries
- track performance by category
- monitor trends over time
- use the site smoothly on desktop and mobile

## Tech Stack
- Frontend: HTML, CSS, JavaScript
- Framework: Flask
- Styling: clean dashboard style
- Charts: Chart.js
- Deployment: local first, scalable later

## Design Direction
- clean and professional
- simple layout
- easy to read
- dashboard-style UI
- responsive design
- clear spacing
- consistent buttons, cards, and tables

## Main Pages
- Dashboard
- KPI Detail Page
- Report Page
- Settings Page

## Components
- top navigation bar
- sidebar menu
- KPI cards
- charts
- filter section
- data table
- status badges
- action buttons

## Coding Rules
- keep code simple and readable
- separate structure, style, and logic clearly
- reuse components when possible
- do not add unnecessary libraries
- make layout responsive
- use clear variable names
- keep file structure organized

## UX Rules
- important numbers must be easy to see
- charts should be simple and not crowded
- users should understand the page quickly
- avoid too much text in one section
- use labels and headings clearly

## File Structure
- app.py
- templates/
- static/
  - css/
  - js/
  - images/

## What to Do When Building
- first create the basic layout
- then build each page section
- then connect charts and data
- then improve styling
- then test responsiveness

## What to Avoid
- messy layout
- too many colors
- unclear chart labels
- repeated code
- large blocks of text
- overcomplicated features too early

## UI Style Rules
- use a white or light background
- use soft shadows for cards
- use rounded corners
- use one main accent color
- keep spacing consistent
- use readable font sizes
- prioritize clarity over decoration
